# 📚 배운 점
- JavaScript는 정적인 HTML 문서에 생명을 불어넣어 동적인 웹 페이지를 만드는 언어임
- 브라우저가 HTML을 DOM이라는 객체 트리 구조로 변환하여 JavaScript가 이를 조작할 수 있게 함
- 변수 선언 시 const를 기본으로 사용하고, 재할당이 필요할 때만 let을 사용하는 것이 권장됨
- var는 호이스팅 문제와 함수 스코프 특성으로 인해 현재는 사용을 권장하지 않음
- DOM 조작의 핵심은 '선택 → 조작'의 순서를 따르는 것
- classList의 toggle() 메서드는 클래스가 있으면 제거하고 없으면 추가하는 편리한 기능
- textContent는 HTML 태그를 완전히 제거하고 순수한 텍스트만 다룸

# 📌 핵심 개념
- **DOM (Document Object Model)**: 웹 페이지를 구조화된 객체로 표현하여 프로그래밍 언어가 페이지 구조에 접근할 수 있게 하는 API
- **블록 스코프**: if, for, 함수 등의 중괄호({}) 내부를 가리키며, 블록 스코프를 가지는 변수는 블록 바깥에서 접근 불가능
- **querySelector vs querySelectorAll**: querySelector는 첫 번째 요소 하나만 반환하고, querySelectorAll은 일치하는 모든 요소를 NodeList로 반환
- **classList 메서드**: add(추가), remove(제거), toggle(토글) 메서드로 클래스를 동적으로 제어
- **호이스팅**: var로 선언된 변수가 선언 위치와 관계없이 스코프 최상단에서 선언된 것처럼 동작하는 현상
- **ECMAScript**: JavaScript의 표준 명세로, JavaScript는 이 표준을 구현한 구체적인 프로그래밍 언어

# 💡 예시 코드

```javascript
// 변수 선언 - const를 기본으로 사용
const userName = '홍길동';
let userAge = 25;  // 재할당이 필요한 경우에만 let 사용

// DOM 요소 선택
const heading = document.querySelector('.heading');
const allContent = document.querySelectorAll('.content');

// HTML 콘텐츠 조작
heading.textContent = '내용 수정';

// 클래스 속성 조작
heading.classList.add('red');
heading.classList.remove('red');
heading.classList.toggle('red');  // 있으면 제거, 없으면 추가

// 일반 속성 조작
const aTag = document.querySelector('a');
aTag.setAttribute('href', 'https://www.naver.com/');
console.log(aTag.getAttribute('href'));
aTag.removeAttribute('href');

// DOM 요소 생성 및 추가
const newH1 = document.createElement('h1');
newH1.textContent = '제목';
const divTag = document.querySelector('div');
divTag.appendChild(newH1);

// 요소 삭제
const pTag = document.querySelector('p');
divTag.removeChild(pTag);

// 스타일 조작
pTag.style.color = 'crimson';
pTag.style.fontSize = '2rem';
pTag.style.border = '1px solid black';
```

# 🛠️ 이슈 & 해결
| 🐞 문제 상황 | 🔍 원인 | 💡 해결 방법 |
|--------------|--------|--------------|
| querySelector로 선택한 요소를 변경했는데 화면에 반영이 안 됨 | 요소를 생성만 하고 appendChild로 문서에 추가하지 않음 | createElement로 생성 후 반드시 appendChild로 DOM 트리에 추가해야 화면에 표시됨 |
| var로 선언한 변수가 선언 전에 사용되어 undefined가 출력됨 | var의 호이스팅 특성으로 선언이 최상단으로 끌어올려짐 | let 또는 const를 사용하면 TDZ(Temporal Dead Zone)로 인해 초기화 전 접근 시 에러 발생 |
| classList로 여러 클래스를 한 번에 추가하려 했으나 방법을 모름 | add() 메서드가 여러 인자를 받을 수 있다는 것을 몰랐음 | classList.add('class1', 'class2', 'class3')처럼 쉼표로 구분하여 여러 클래스 동시 추가 가능 |
