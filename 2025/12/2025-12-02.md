# 📚 배운 점
- Vue의 SFC는 `<template>`, `<script setup>`, `<style scoped>` 세 부분으로 구성되며, 하나의 .vue 파일에 HTML, JavaScript, CSS를 모두 작성할 수 있다
- Vite는 빠른 개발 서버와 빌드 기능을 제공하는 Vue의 공식 프론트엔드 빌드 도구이다
- 컴포넌트는 재사용 가능한 독립적인 UI 블록으로, 레고 블록처럼 조립하여 페이지를 구성한다
- Virtual DOM을 통해 실제 DOM 조작을 최소화하여 성능을 향상시킨다
- package.json은 프로젝트 설계도, package-lock.json은 정확한 버전 기록, node_modules는 실제 패키지 저장소 역할을 한다

# 📌 핵심 개념
- **SFC (Single-File Components)**: 컴포넌트의 템플릿, 로직, 스타일을 하나의 .vue 파일로 관리하는 Vue의 개발 방식
- **Vite**: 프론트엔드 빌드 도구로 빠른 개발 서버와 HMR(Hot Module Replacement) 제공
- **NPM**: Node Package Manager로 JavaScript 패키지를 설치하고 관리하는 도구
- **Virtual DOM**: 실제 DOM의 가상 복사본으로, 변경사항을 계산한 후 최소한의 업데이트만 실제 DOM에 적용
- **Component 계층 구조**: App.vue(부모) - MyComponent.vue(자식) 형태로 트리 구조를 형성
- **scoped 스타일**: `<style scoped>`를 사용하여 해당 컴포넌트에만 스타일이 적용되도록 범위 제한

# 💡 예시 코드

```vue
<!-- MyComponent.vue -->
<template>
  <div>
    <h2>MyComponent</h2>
    <p>재사용 가능한 컴포넌트입니다</p>
  </div>
</template>

<script setup>
import { ref } from 'vue'
const msg = ref('Hello World!')
</script>

<style scoped>
.greeting {
  color: red;
}
</style>
```

```vue
<!-- App.vue -->
<template>
  <h1>App.vue</h1>
  <MyComponent />
</template>

<script setup>
// '@'는 'src/' 경로를 의미하는 약어
import MyComponent from '@/components/MyComponent.vue'
</script>
```

```bash
# Vue 프로젝트 생성
$ npm create vue@latest

# 프로젝트 폴더로 이동
$ cd vue-project

# 패키지 설치
$ npm install

# 개발 서버 실행
$ npm run dev
```

# 🛠️ 이슈 & 해결
| 🐞 문제 상황 | 🔍 원인 | 💡 해결 방법 |
|--------------|--------|--------------|
| .vue 파일을 브라우저에서 직접 실행할 수 없음 | 브라우저는 .vue 파일 형식을 이해하지 못함 | Vite 빌드 도구를 통해 컴파일 후 실행 |
| 컴포넌트 스타일이 다른 컴포넌트에 영향을 미침 | `<style>` 태그에 scoped 속성 누락 | `<style scoped>`를 사용하여 스타일 범위 제한 |
| DOM 직접 조작 시 예측 불가능한 렌더링 발생 | Vue의 Virtual DOM 시스템과 충돌 | ref()와 Lifecycle Hooks를 사용한 간접 접근 |
